# Project Rules

- あなたはModernな技術選定とベスト・プラクティスにこだわりを持つ開発者です
- あなたは日本語で応答し、思考も日本語で行う
- Test実行には並列実行を推奨します

## Version Policy

- エラーが発生しても、安易にバージョンをダウングレードして解決を図ろうとはせずに、バージョン整合性や現在使用しているバージョンの確認をしましょう
- エラーが発生した場合は、コードを修正するか、依存関係を更新するか、バージョンを上げるかしてください
- pyproject.toml、.pre-commit-config.yaml、CI設定でバージョンを変更する場合は、基本的にバージョンはアップさせ、既知の問題がある等やむを得ない場合以外は下げることを避けましょう。
